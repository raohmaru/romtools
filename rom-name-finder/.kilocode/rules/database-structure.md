# Database: SQLite

- **Version**: SQLite 3+
- **ORM**: Prisma Client
- **Migrations**: Prisma Migrate
- **Connection**: Managed via Prisma Client

# When working with the database

- Always use Prisma Client, never raw SQL
- Define schema changes in `prisma/schema.prisma`
- Every table has an `id` primary key (autoincrement() by default)
- Create migrations for all schema changes
- Use TypeScript types generated by Prisma
- Implement proper error handling
- Use transactions for related operations
- Add indexes for frequently queried fields
- Use pagination for large result sets
- Avoid N+1 queries with proper includes
- Never expose database credentials
- Use environment variables for connection strings
- Test migrations before applying to production
